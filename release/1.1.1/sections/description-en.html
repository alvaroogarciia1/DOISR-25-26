Digimon Ontology: Description back to ToC

The ontology models Digimon entities and their relationships through a hierarchy of classes and properties.

The central concept of the ontology is the class Digimon, which is specialized into subclasses representing the different Digimon types: VirusDigimon, VaccineDigimon, DataDigimon and FreeDigimon.

Each Digimon is linked to:

  • a Stage,
  • an Attribute,
  • one or more Moves,
  • Skills and Support Skills,
  • and possible Digivolution relationships.

Digivolutions are modeled as n-ary relations through the class Digivolution, which connects source Digimon, target Digimon and Digivolution Requirements.

The ontology also models numerical Digimon statistics such as HP, SP, attack, defense, intelligence, speed and memory through datatype properties.

The ontology was validated using SHACL constraints and evaluated using OOPS! and Widoco tools.
